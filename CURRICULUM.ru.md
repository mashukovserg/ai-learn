# Учебная программа AI: путь для начинающих

## Путь: Основы AI
**Цель:** Понять экосистему, геополитику и базовые механизмы современных LLM.

### Модуль 1: Современный ландшафт AI
- [x] **Комната 101: Ландшафт LLM** — 13 заданий
  - *Темы:* Original Gangster (OpenAI), Лидерборды (ELO/Chatbot Arena), Геополитика (США vs Китай vs ЕС), Суверенный AI, рамка выбора модели, зрелость SDK-интеграции.
- [x] **Комната 102: Как мыслят LLM** — 8 заданий (включая сортировку)
  - *Темы:* Токены, Контекстные окна, Предсказание следующего токена, Температура.
- [x] **Комната 103: Основы промптинга** — 5 заданий
  - *Темы:* Системные промпты, Few-shot/Zero-shot, Борьба с галлюцинациями, Итерация промптов.

### Модуль 2: Мультимодальность
- [x] **Комната 201: Нативная мультимодальность** — 3 задания
  - *Темы:* Зрение (Image Patches), Аудио (токены спектрограммы), Ассистенты реального времени.
- [x] **Комната 202: Research & Grounding** — 6 заданий
  - *Темы:* Основы RAG, качество поиска, цитирование источников и калибровка доверия.

### Модуль 3: Архитектура и адаптация
- [x] **Файн-тюнинг и адаптация** — 12 заданий (категоризация, сценарий, ментор)
  - *Темы:* LoRA, QLoRA, полный файн-тюнинг vs промптинг vs RAG, подготовка данных, переобучение, катастрофическое забывание, дерево решений.

### Модуль 4: Продвинутые AI-системы
- [x] **AI-агенты** — 6 заданий (Обогащено)
  - *Темы:* Паттерны агентности, цикл ReAct, MAS (многоагентные системы), использование инструментов.
- [x] **AI RAG (Retrieval-Augmented Generation)** — 6 заданий (Обогащено)
  - *Темы:* Чанкинг, Эмбеддинги, Векторный поиск, Аугментация, Реранкинг.
- [x] **Безопасность ИИ (AI Security)** — 6 заданий (Обогащено)
  - *Темы:* Промпт-инъекции, Джейлбрейки, паттерны защиты (Dual-LLM).
- [x] **ИИ-исследователь (AI Research)** — 5 заданий (Обогащено)
  - *Темы:* Автоматизированный поиск, синтез, проверка гипотез.

### Модуль 5: Практика
- [x] **Prompt Evaluation & Evals** — 6 заданий
  - *Темы:* Наборы оценки, метрики, release gates для LLM-фич.
- [x] **AI для создания изображений** — 6 заданий
  - *Темы:* Промпты, композиция, негативные промпты, управление seed, коммерческое использование.

## Путь: Идеи и споры об ИИ
**Цель:** Понять основные концептуальные дебаты в AI-сообществах и трансформировать их в практические продуктовые решения.

- [x] **ChatGPT moment** — 6 заданий
  - *Темы:* Психология интерфейса и массовая адаптация, вирусные петли дистрибуции, экономика поиска в режиме "Code Red", продуктовые и governance-импликации.
- [x] **Гипотеза масштабирования** — 5 заданий
  - *Темы:* Три столпа (вычисления/данные/параметры), Эмерджентные способности, Оптимальность по Chinchilla.
- [x] **Сингулярность в AI-дебатах** — 6 заданий
  - *Темы:* Аргументы за/против, структура сравнения лагерей, фрейминг рисков, операционные импликации для команд.
- [x] **История AI** — 4 задания (включая таймлайн)
  - *Темы:* Таймлайн AI, ключевые вехи, ключевые голоса в дискуссии о рисках ИИ, от подраздела Computer Science до глобального явления.
- [x] **Сингулярность в AI-дебатах** — 6 заданий
- [x] **Пост-ChatGPT эпоха (Post-ChatGPT Era)** — 6 заданий
  - *Темы:* Компромиссы API-доступа, динамика экосистем open-weight, модели рассуждений (o1), геополитика вычислений.

## Путь в rollout: Agent Coding (Агентная разработка)
**Цель:** Научить доводить разработку с coding-агентами до production-качества через строгие инженерные циклы, проверяемые quality-gates и командные протоколы.

### Модуль A: База Agent Coding
- [x] **AC-101: База Agent Coding** (`agent-coding-foundations`) — 10 заданий; постановка задач, скорость итераций, критерии приемки, guardrails, rollback.
- [ ] **AC-102: Prompt-контракты для coding-агентов** — спецификации, ограничения, контракт ответа.
- [ ] **AC-103: Паттерны multi-agent разработки** — декомпозиция, контекстная дисциплина, стратегия веток.

### Модуль B: Цикл сборки и качество
- [ ] **AC-201: Навигация по репозиторию с coding-агентами** — семантическое исследование и безопасный рефакторинг.
- [ ] **AC-202: Agentic тестовый цикл** — test-first промпты, регрессии, triage сбоев.
- [ ] **AC-203: Agentic UI Delivery** — адаптивная верстка, доступность, консистентность компонентов.

### Модуль C: Доставка и командная операционка
- [ ] **AC-301: Доставка agentic-фич с guardrails** — policy-проверки, red-team промпты, release-gates.
- [ ] **AC-302: Контроль стоимости и задержки для агентов** — маршрутизация, кеширование, бюджетные границы, observability.
- [ ] **AC-303: Командные протоколы Agent Coding** — стандарты ревью, handoff, журналы решений.

### Модуль D: Продакшн и капстоун
- [ ] **AC-401: Инцидентные playbook для agentic-фич** — rollback, hotfix, postmortem.
- [ ] **AC-402: Data/Eval-операции для agent-команд** — eval-наборы, drift-контроль, quality-дашборды.
- [ ] **AC-499: Agent Coding Capstone** — полный цикл поставки фичи от брифа до production-релиза.

## Примечания о доступности
- Все комнаты обслуживаются динамическим маршрутом `src/app/[lang]/rooms/[id]/page.tsx`. Теоретический контент находится в `src/components/theory/`.
- Прогресс сохраняется через API бэкенда при авторизации, с fallback на localStorage для гостей (хук `useProgress`).
- На 2026-03-24 обновлён UI-базис под режим чтения: уточнены типографические стеки, для длинных теоретических блоков добавлен `reading-prose`, ритм отступов на уровне оболочки унифицирован через `content-shell`.
