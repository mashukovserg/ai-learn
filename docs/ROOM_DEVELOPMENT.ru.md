# Развитие комнат: логика, чеклист и очередь

Последняя сверка с реестром: **12.07.2026** (40 комнат в `ROOMS_METADATA`, подключено 39; у `prompt-evals` нет файла заданий).

Единый авторитетный документ о том, **какую комнату строить следующей** и **как правильно добавлять комнату**. Заменяет список идей в `ROOMS_IDEAS.md` (теперь заглушка-указатель) и краткий 4-шаговый список «Adding a New Room» в `CLAUDE.md`. Гейты, на которые он ссылается, живут в [`AGENTS.md`](AGENTS.md) — этот документ их не заменяет.

## Как агентам пользоваться документом

1. Работа берётся из **очереди** сверху вниз внутри старшего открытого приоритета. Не начинать P2-комнату, пока открыт пункт P0.
2. Каждая идея новой комнаты попадает в очередь (см. «Приём идей») **до** написания кода — включая идеи владельца проекта: сначала строка в очереди, потом сборка.
3. Когда комната отгружена, в той же сессии: перенести её строку в «Отгружено», обновить дату сверки выше, выполнить синхронизацию документации из чеклиста.
4. Если отклонились от очереди (владелец попросил другое) — всё равно записать собранную комнату в «Отгружено» с пометкой.

## Логика приоритетов

Полосы работы, по порядку:

- **P0 — сломанное или обещанное.** Всё, что есть в `ROOMS_METADATA` без файла заданий (комната сломана в проде), и отслеживаемые дефекты данных, блокирующие тестовый гейт (punch-list'ы в `BACKLOG.md`). Ничего нового не строится, пока в проде есть сломанная комната.
- **P1 — обязательства роадмапа.** Комнаты с явным коммитом в [`ROADMAP_3M.ru.md`](ROADMAP_3M.ru.md). Сегодня: шесть комнат Agent Coding (каноническая нумерация ниже).
- **P2 — тонкие места.** Категории с двумя комнатами (сегодня: Безопасность, Открытые модели) и модули CURRICULUM с двумя комнатами (сегодня: Модуль 2 Мультимодальность, Модуль 5 Практика — Модули 3/6 растут за счёт уже запланированных строк очереди). Тонкое место — аргумент, а не приказ: выбирайте то, чья аудитория пересекается с самым сильным существующим путём.
- **P3 — бэклог идей.** Всё остальное в очереди.

### Эвристики выбора (внутри полосы)

1. **Сначала укреплять, потом расширять.** Пополнение существующего пути/категории предпочтительнее открытия новых. **Новая категория — только при 2+ запланированных комнатах** (прецедент: «Открытые модели» открылись парой).
2. **Паттерн кластера.** Пара «широкое интро + углубление»: `local-models-101` + `llama-3-1-8b`, `research-ai-era` + `ai-literature-reviews`, планируемая `prompt-evals` + `evalops-basics`. Если идея не влезает в одну комнату — проектировать кластер сразу и отметить в очереди.
3. **Проверка на протухание фактов.** Комнаты строятся на устойчивых концептах (механизмы, компромиссы, воркфлоу). Скоропортящаяся конкретика — цены, цифры бенчмарков, модели сезона — только в callout'ах теории с датой и inline-цитированием (см. `AGENTS.md` → Inline citation format), никогда в ответах заданий. Конкретно: (a) используем **семейства моделей, а не версии** (GPT / Claude / Gemini / Llama / Mistral / Qwen / DeepSeek); конкретные версии — только в датированных callout'ах со ссылками или в подписях цитат. (b) **Никаких категоричных рейтингов «лучший = такой-то игрок»**, и тем более не в правильном ответе задания — формулируем как «часто в числе лидеров» и отсылаем ученика к собственному eval-набору. Прецедент: рефактор устойчивости `llm-landscape` (16.07.2026).
4. **Баланс сложности.** Перед выбором проверить распределение сложности целевого пути; пути из одних Beginner-комнат нужнее Intermediate-капстоун, чем пятое интро.
5. **pathIds на этапе идеи.** Каждая строка очереди называет целевые `pathIds`. Комната без пути невидима на страницах путей — см. правила когерентности ниже.

## Канонический чеклист добавления комнаты

Полная процедура. Короткий список в `CLAUDE.md` — конспект шагов 3–6; авторитет — этот список.

1. **Проверка на дубли.** Grep по `src/components/theory/` и `src/data/rooms/tasks/` по ключевым терминам темы (EN + RU). При существенном пересечении — расширить существующую комнату или спроектировать кластер, а не дублировать.
2. **Размещение.** Назначить `pathIds` (обязательно, ≥1 разблокированный путь), категорию (существующую, если не выполнено правило двух комнат) и модуль CURRICULUM.
3. **Метаданные** в `src/data/rooms/metadata.ts`, рядом с кластерным соседом, если он есть.
4. **Теория** в `src/components/theory/<RoomName>Theory.tsx`: 5 глав, обе локали, одна колонка. Гейты: глубина главы (≥120 слов / ≥2 абзаца на язык), без иконок в заголовках, Anti-Vibecode, формат цитирования, оформление цитат/callout'ов. **Перед отгрузкой спросить владельца, какие термины обернуть в `<Term>`.**
5. **Задания** в `src/data/rooms/tasks/<room-id>.ts`: 8–10 заданий, ≥3 типов, правило микса (≥1 `sorting` или `mentor`), валидационный гейт по типам, последовательные ID с 1. Ключи `correctMapping` в `categorize` равны `items[].en` дословно.
6. **Регистрация в ДВУХ местах:** `src/data/rooms/tasks/index.ts` (импорт + запись в `ROOM_TASKS`) **и** `THEORY_COMPONENTS` в `src/app/[lang]/rooms/[id]/page.tsx`. Забытое второе место молча рендерит заглушку.
7. **Глоссарий.** Новые концепты — записи в `src/data/glossary.ts` (обе локали) и `<Term>`-обёртки в теории.
8. **Обложка.** Программная через `RoomCover`: добавить иконку комнаты в `ICON_MAP` в `src/components/roomVisuals.ts`, если её нет; hue тона обложки должен быть замаплен в saas-блоке `globals.css`. Опциональный арт: промпт в `IMAGE_GENERATION_PROMPTS.md` с указанием целевого файла.
9. **Проверка.** `npm run check-all` чисто; `npm run test` — без новых падений сверх отслеживаемого списка дефектов; пройти каждое задание в обеих локалях на `http://localhost:3000` (прогресс должен достичь 100%).
10. **Синхронизация документации** (та же сессия, по Completion checklist из `AGENTS.md`): журнал в `BACKLOG.md`, веха и счётчик комнат в `PROGRESS.md`, инвентарь комнат в `AGENTS.md`, запись модуля в `CURRICULUM.md`, счётчики в `README.md` — плюс строка очереди здесь переезжает в «Отгружено».

## Правила когерентности учебной программы

1. **`pathIds` обязательны** для каждой комнаты. Историческая справка: 12.07.2026 девять старых комнат были сиротами без пути (невидимы на всех страницах путей, хотя CURRICULUM числил их в модулях) — исправлено пакетно. Не воспроизводить этот дрейф.
2. **Запертые пути не считаются.** `intermediate` имеет `unlocked: false` — комната, назначенная на него, обязана нести и разблокированный путь, иначе она недостижима со страниц путей.
3. **Минимум категории.** Новая категория требует 2+ запланированных комнат (естественная пара — интро + углубление).

## Приём идей

Чтобы предложить комнату: добавьте одну строку в таблицу нужного приоритета со всеми пятью колонками. Если идея пришла из разговора с владельцем — строка добавляется в той же сессии. Если идея дублирует существующую комнату — расширяйте её (шаг 1 чеклиста). Идеи-фичи, не являющиеся комнатами (лаборатории, механики, артефакты), получают строку, только если блокируют работу над комнатами; иначе их место в `BACKLOG.md` / `IDEAS_NEXT_STEPS.md`.
