# Design & Product References

Разборы внешних сайтов и продуктов, на которые мы ориентируемся при развитии платформы. Один файл — один референс.

## Зачем эта папка

Разбор референса делается один раз и фиксируется здесь, чтобы: (а) не пересматривать сайт заново при каждом дизайн-решении, (б) агенты могли сверяться с уже принятыми решениями «берём / не берём», (в) извлечённые токены (палитра, шрифты, CSS-приёмы) были под рукой в сыром виде.

## Конвенция

1. Имя файла — слаг домена: `re-alisa-sh.md`, `linear-app.md`.
2. Обязательные секции: шапка (URL, дата съёма), извлечённые токены/приёмы (фактаж), разбор, решения «что берём / что не берём» с привязкой к нашим гейтам (`../AGENTS.md`), статус применения.
3. Решения «берём/не берём» сверяются с Anti-Vibecode Frontend Gate и правилами дизайна в `../AGENTS.md` — референс не отменяет гейты.
4. Язык — русский (язык дизайн-обсуждений в проекте); зеркала `.ru.md` для этой папки не ведутся.
5. При реализации приёма из референса — обновить его статус применения в файле референса в той же сессии.
