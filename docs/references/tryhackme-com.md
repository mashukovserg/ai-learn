# Референс: tryhackme.com (TryHackMe)

- **URL:** https://tryhackme.com
- **Дата съёма:** 2026-07-12 (лендинг — визуально + computed styles; roadmap и страница комнаты — текстовая структура; продуктовый UI за логином — стили не отдаются без авторизации)
- **Что это:** платформа обучения кибербезопасности, наш исходный образец («TryHackMe-style» в описании проекта): комнаты → задания → пути → геймификация
- **Зачем нам:** сверка с оригиналом — что из их системы прогрессии и UX-паттернов у нас уже есть, чего не хватает, что сознательно не берём

### Структура прогрессии (с roadmap-страницы)

Три уровня + сертификация:

```
Карьерный трек (Security Analyst / Pentester / Security Engineer / AI)
  └─ Path (SOC Level 1, Jr Penetration Tester…) + difficulty-бейдж (easy/intermediate/hard)
      └─ Modules → Rooms
```

Плюс: счётчики масштаба на витрине («1.3K+ Hands-on Labs», «500+ Challenges»), карьерный квиз («Not sure which career path? Take our career quiz»).

## Разбор

THM выигрывает не эстетикой (дизайн — обычный SaaS с маскотом-космонавтом), а **системой прогрессии**: короткая петля «ответил → зелёное → очки» вложена в среднюю петлю «комната → путь» и длинную «путь → сертификат с кодом экзамена». Каждый уровень имеет видимую награду. Социальное доказательство встроено в мета-данные комнаты (счётчик прошедших) и в витрину (счётчики контента). Бренд-сигнал «мы — про хакеров» несут шрифт Ubuntu, терминальные скриншоты и маскот, а не тёмная тема сама по себе — лендинг наполовину светлый.

Контраст с [re-alisa-sh.md](re-alisa-sh.md) показателен: alisa.sh достигает эффекта вычитанием формы, THM — наращиванием структуры. Для нас актуальны оба урока: сдержанная форма + сильная прогрессия.

## Что берём для платформы (совместимо с нашими гейтами)

| Приём | Как применить у нас |
|---|---|
| Счётчики масштаба на витрине | Дашборд/каталог: «40 комнат · N заданий · M путей» — дёшево, даёт ощущение массы контента |
| Счётчик прошедших у комнаты | После аналитики (M3-3 роадмапа): «N прошли эту комнату» в мета-строке комнаты — социальное доказательство |
| Коды путей как у сертификаций | Пути получают короткие коды (AIF-1, AC-1…) в моно-шрифте — серьёзность + ложится на терминальную тему и моно-лейблы из re-alisa-sh |
| Маска формата ответа | В `input`-заданиях показывать шаблон ответа (`___ ___` / `**-**`) — снижает фрустрацию угадывания формата |
| Квиз-роутер пути | «Не знаешь, с чего начать — пройди квиз» → рекомендация пути; у нас уже есть professions-страница как основа |
| Интерактивные вставки в теории | Подтверждение нашего курса: PromptPlayground в теории = их «Task includes website»; расширять этот паттерн |

## Что не берём и почему

- **Лайм-CTA и цветовую агрессию** — у нас emerald-язык и anti-vibecode gate; конверсионная кричащесть THM нам не нужна.
- **Ubuntu-шрифт** — их бренд-сигнал; наш «терминальный» сигнал уже несут mono-элементы, текст остаётся Inter/SF Pro (решение владельца 2026-07-09).
- **Premium-гейтинг комнат** — не наша модель; у нас гейтинг только уровневый (locked-путь).
- **Маскота** — отдельное бренд-решение, не сейчас; обложки остаются программными (roomVisuals).
- **Split-view теория|VM** — у нас нет виртуалок; наш аналог — инлайн-плейграунды в теории.
- **Светлый маркетинговый лендинг** — у нас продукт и есть витрина, отдельного лендинга нет.

## Мета-урок

Геймификация держится на вложенных петлях вознаграждения с видимым «следующим уровнем» на каждом горизонте: ответ → комната → путь → сертификат. У нас короткая петля есть (очки, конфетти), средняя есть (прогресс комнаты/пути), длинная — нет (ни сертификатов, ни кодов, ни финального артефакта пути). Это самый большой структурный разрыв между нами и оригиналом.
