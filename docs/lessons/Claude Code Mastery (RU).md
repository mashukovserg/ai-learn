# Claude Code Mastery: От ИИ-чата к Агентному инженеру

Добро пожаловать в путешествие по миру Claude Code — инструмента, который меняет само определение «программирования с ИИ». Мы не просто будем учиться писать промпты; мы научимся управлять умным автономным стажером, у которого есть доступ к вашему терминалу, файлам и инструментам сборки.

## Дорожная карта (Roadmap)
1. **От калькулятора к стажеру:** Революция агентного кодинга.
2. **Первое рукопожатие:** Установка и настройка.
3. **Сердце машины:** Агентный цикл (The Loop).
4. **Штурман или Автопилот?** Режимы Normal и Plan.
5. **Машина времени:** Управление сессиями и откат ошибок.

---

### Комната 1: От калькулятора к стажеру
#### Революция агентного кодинга

**Концептуальная аналогия:** 
Представьте разницу между умным калькулятором (обычный ChatGPT или Claude в браузере) и живым стажером. Калькулятору нужно *принести* данные, нажать кнопки, а потом *самому* переписать результат в тетрадь. Стажеру же можно сказать: «Проверь, почему тесты в папке `auth` падают, исправь их и запусти заново». Стажер сам найдет файлы, сам запустит команды и сам проверит итог. 

**Зачем это нужно (Justification):**
Традиционные ИИ-помощники требуют от вас огромного количества «копипасты». Вы копируете ошибку из терминала в чат, копируете код обратно... Это выматывает и убивает фокус. Claude Code — это **агентный инструмент**. Он «видит» вашу кодовую базу целиком, может выполнять команды в терминале и редактировать файлы напрямую.

**Что это значит на практике?**
- Он читает файлы (контекст).
- Он запускает тесты и сборку (действие).
- Он видит ошибки в реальном времени (обратная связь).
- Он исправляет их в цикле, пока не добьется успеха.

### Комната 2: Первое рукопожатие
#### Установка и настройка рабочего места

**Концептуальная аналогия:**
Прежде чем стажер начнет работать, ему нужен пропуск в здание, стол и компьютер. В мире Claude Code это — установка CLI и авторизация.

**Дорожная карта установки:**
1. **Системные требования:** Вам нужно минимум 4 ГБ ОЗУ и установленный Git (на Windows — Git for Windows).
2. **Установка:** Обычно это одна команда в терминале (через `npm` или `curl`).
3. **Авторизация:** Команда `claude auth login` связывает ваш локальный терминал с облачным мозгом Anthropic.

**Важный нюанс (Why it matters):**
Claude Code работает прямо в вашем терминале. Это дает ему «суперспособность» — использовать ваши локальные инструменты (grep, npm, pytest, go). Вы не просто общаетесь с облаком, вы расширяете свой терминал искусственным интеллектом.

### Комната 3: Сердце машины
#### Агентный цикл (The Loop)

**Концептуальная аналогия:**
Как работает мозг опытного инженера? 
1. *«Так, что тут происходит?»* (Сбор контекста).
2. *«Попробую-ка я вот это...»* (Действие).
3. *«О, сработало? Или упало?»* (Проверка).
Claude Code делает ровно то же самое в бесконечном цикле, пока задача не будет решена.

**Три фазы цикла:**
1. **Gather Context (Сбор контекста):** Claude читает файлы, ищет определения функций, сканирует структуру проекта.
2. **Take Action (Действие):** Он пишет код, создает файлы или запускает команды сборки.
3. **Verify Results (Проверка):** Он анализирует вывод терминала. Если тесты упали — он возвращается на шаг 1, но уже с новым знанием об ошибке.

**Совет (Pro-tip):**
Вы всегда можете прервать этот цикл (Ctrl+C). Если вы видите, что «стажер» пошел не в ту сторону, просто остановите его и дайте уточняющую подсказку. Это интерактивный процесс, а не «черный ящик».

### Комната 4: Штурман или Автопилот?
#### Режимы Normal и Plan

**Концептуальная аналогия:**
Когда вы едете на машине, вы можете ехать по навигатору, поворачивая на каждом перекрестке (Normal), а можете сначала изучить карту всего маршрута и утвердить его (Plan).

**Два режима управления:**
- **Normal Mode (Автопилот):** Claude сразу приступает к делу. Он будет спрашивать разрешение только на выполнение критических команд. Идеально для мелких правок и быстрых вопросов.
- **Plan Mode (Штурман):** Claude сначала создает подробный план действий в формате Markdown. Вы читаете его, вносите правки и только после нажатия «Approve» он начинает менять код.

**Зачем выбирать Plan Mode?**
Для сложных задач (например, «перепиши всю логику авторизации с JWT на Sessions») риск сломать проект велик. Plan Mode позволяет вам быть «архитектором», который утверждает чертежи, прежде чем «строитель» (Claude) возьмет молоток.

### Комната 5: Машина времени
#### Управление сессиями и откат ошибок

**Концептуальная аналогия:**
Вспомните видеоигры. Перед сложным боссом вы всегда сохраняетесь (Save). Если вы проиграли — вы просто загружаетесь (Load). Сессии в Claude Code — это ваши точки сохранения.

**Основные команды «Машины времени»:**
- `/resume`: Позволяет вернуться к предыдущему разговору (сессии). Помните: Claude хранит историю ваших сессий 30 дней.
- `/rewind`: Самая мощная команда. Если Claude внес 10 изменений и последние два сломали проект — вы можете «отмотать» время назад к моменту, когда всё работало.
- `/compact`: Если память сессии переполнена (слишком много контекста), эта команда сожмет историю, оставив только самое важное.

**Почему это важно (Justification):**
Многие боятся ИИ-агентов, потому что «они всё испортят». Машина времени убирает этот страх. Вы всегда в безопасности: любое действие можно отменить, а к любому контексту — вернуться.
