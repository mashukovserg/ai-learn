# Как мыслят LLM: Внутри черного ящика

**Уровень:** Начинающий  
**Длительность:** 1.5 ч

Связанные темы: [[ChatGPT Moment (RU)]], [[LLM Landscape (RU)]]

## Глава 1: Токенизация и BPE

Нейросети не читают текст буквами или словами. Они разбивают его на токены с помощью алгоритма BPE (Byte Pair Encoding). Токен — это статистически значимый фрагмент текста. Понимание токенов критично для контроля затрат на API и понимания ограничений модели (например, почему ИИ плохо считает буквы в словах).

## Глава 2: Предсказание следующего токена

Весь интеллект модели — это цикл предсказания. На основе всей истории диалога модель вычисляет вероятности для каждого токена из своего словаря (обычно около 100,000 вариантов). Это локальный процесс: модель не знает конца предложения, пока не дойдет до него.

## Глава 3: Температура и Сэмплирование

Температура — это способ управления случайностью. При T=0 модель всегда выбирает самый вероятный токен (Greedy Search), что идеально для кода. При T=0.7 модель начинает "рисковать", делая текст более естественным и человечным.

## Глава 4: Контекстное окно и Память

Контекстное окно — это объем данных, которые модель может "видеть" одновременно. Использование KV-Cache позволяет экономить ресурсы при генерации длинных текстов, но даже самые мощные модели подвержены эффекту "Lost in the Middle", теряя фокус в середине огромных документов.

## Глава 5: Почему возникают галлюцинации

Галлюцинации — это прямое следствие вероятностной природы моделей. Если у ложного, но правдоподобно звучащего ответа вероятность выше, чем у сухого факта, модель может выбрать ложь, особенно при высоких значениях температуры.
