# Роадмап для агентов на 3 месяца (16.05.2026 → 15.08.2026)

План работ для coding-агентов в этом репозитории. Каждый пункт рассчитан на наблюдаемый темп поставки (~1 значимый блок за 1–2 недели) и содержит собственные критерии приёмки. Внутри месяца работаем сверху вниз; не начинаем следующий месяц, пока открыт 🔴-пункт текущего.

Сопутствующие документы: `AGENTS.md` (обязательные гейты), `BACKLOG.md` (списки задач, журнал работ), `TESTING.md` (стратегия тестирования и триаж дефектов), `ROOMS_IDEAS.md` (идеи комнат).

## Как агентам использовать этот роадмап

1. Берём самый верхний незакрытый пункт текущего месяца.
2. Перед правками перечитываем соответствующие гейты в `AGENTS.md` (валидация заданий, нумерация ID, паритет локалей, матрица документации).
3. Доводим пункт до конца: код + обе локали + синхронизация документации + запись в `BACKLOG.md` с пометкой `(by <имя агента>)`.
4. Отмечаем пункт здесь в той же сессии.
5. Перед завершением запускаем `npm run check-all` (и `npm run test`, когда гейт включён — см. M1-3).

## Сквозные правила (повтор из `AGENTS.md` — действуют всегда)

1. **Паритет локалей:** каждая пользовательская строка отгружается в `en` и `ru` в одной сессии.
2. **Гейт валидации заданий:** проверять проходимость каждого задания в обеих локалях; правила по типам — в `AGENTS.md`.
3. **Нумерация ID заданий:** последовательные целые с 1, без пропусков и дублей.
4. **Матрица документации:** `README.md`, `PROGRESS.md`, `CURRICULUM.md`, `BACKLOG.md`, `DEPLOYMENT.md` (+ зеркала `.ru.md`) обновляются в той же сессии, что и изменение.
5. **Журнал работ:** каждый завершённый блок фиксируется в `BACKLOG.md` → Completed с пометкой `(by <имя агента>)`.
6. **Стилевые гейты:** правила anti-vibecode, без иконок перед заголовками, глубина главы ≥120 слов / ≥2 абзаца на язык, запрещённые шаблонные формулировки — см. `AGENTS.md`.
7. **Доступность фронтенда:** для любой coding-задачи обеспечить работу `http://localhost:3000`; переиспользовать живой сервер, не запускать второй.
