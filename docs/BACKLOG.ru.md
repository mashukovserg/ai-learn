Причина оказалась структурной, а не промахом с одной комнатой: `getRoomsByPath` возвращал `ROOMS_METADATA.filter(...)`, то есть порядок пути = **позиция в массиве** — след того, когда комнату добавили, а не учебная последовательность. Простой перестановкой массива это не чинится — `mcp-tool-ecosystems` входит в два пути и нуждается в разной позиции в каждом, а один индекс массива этого выразить не может. **Теперь порядок принадлежит пути**: в `PathMetadata` добавлено `roomIds: string[]`, и для всех 5 путей прописана осознанная последовательность (agent-coding: модуль A → как цикл работает на практике (Claude Code loop, CLI, контекст-инжиниринг) → циклы качества модуля B → продвинутый хвост; beginner: концепции → практика → прикладное → Advanced в конце; ideas-history: хронологически, scaling перед singularity).
