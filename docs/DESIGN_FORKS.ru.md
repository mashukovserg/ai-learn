# Развилки дизайна — намеренно открытые решения

> **Статус: проект всё ещё ищет оптимальный дизайн.**
> Всё в этом файле — **открытая развилка**, а не принятое окончательно решение. «Текущий выбор»
> здесь означает *что подключено сегодня*, а не *что решено навсегда*. Проигравшие варианты
> сохраняются намеренно, вместе с готовыми к вставке значениями, чтобы откат стоил минут, а не
> раскопок.

English mirror: [`DESIGN_FORKS.md`](DESIGN_FORKS.md).

## Правила для агентов (обязательные)

1. **Не считайте текущий выбор финальным.** Если в задаче сказано «сделай терминал как X» — это
   ход по развилке, а не исправление бага.
2. **Не удаляйте проигравший вариант.** Переносите его в «Альтернативы» вместе с полным набором
   токенов и компромиссом, из-за которого он проиграл. Смысл файла в том, чтобы откат оставался
   дешёвым.
3. **Меняете выбор — обновите этот файл в той же задаче**: новый текущий выбор, прежний переведён
   в альтернативы, дата и одна строка *почему*. Тем же коммитом.
4. **Каждый пресет должен быть полным.** Пресет обязан задавать *все* токены, которые потребляет
   компонент. Неполный пресет тихо ломает утилиты (например, `text-term-blue` разрешится в пустоту).
5. Развилки — решения по дизайну, а не по контенту: обязательные контентные гейты из `AGENTS.md`
   (двуязычие, решаемость, глубина глав) действуют при любом варианте развилки.

## Развилка 1 — дизайн терминала

**Контекст.** `src/components/Terminal.tsx` — элемент дизайн-системы, сейчас отрисовывается в
**11 комнатах**. Весь его вид задаётся токенами `--color-term-*` плюс `--font-term` в
`src/app/[lang]/globals.css` (`@theme`). **Смена дизайна = замена одного блока токенов.** Без
правок компонента и без правок комнат.

**Инвариант для всех вариантов:** терминал остаётся тёмным в *обеих* темах — `--color-term-*`
нельзя переопределять в блоке `[data-theme="saas"]`. Терминал должен выглядеть терминалом и на
светлом UI. Любой новый пресет обязан сохранять это свойство.

**Токены, которые потребляет компонент:** `bg`, `head`, `line`, `text`, `dim`, `prompt`, `red`,
`blue`, `cyan`, `yellow` (+ `--font-term`). Тона в API маппятся так: `ok`→prompt, `bad`→red,
`dir`→blue, `link`→cyan, `warn`→yellow, `dim`→dim.

> **Сравнить варианты глазами: [`terminal-picker.html`](terminal-picker.html)** — откройте в
> браузере. Переключает все пресеты ниже, шрифт терминала (развилка 2b) и его размер, а также фон
> страницы между тёмной и светлой темами сайта, чтобы инвариант «остаётся тёмным в обеих темах»
> можно было проверить визуально. Рисует реальную сессию из AC-201 (с кириллицей — на неё стоит
> смотреть при выборе шрифта) плюс палитровый терминал, задействующий `dir`/`link`/`warn`, и выдаёт
> готовый к вставке блок токенов. Оценивайте пресет по реальному выводу, а не по хексам.

### Текущий выбор — Ubuntu GNOME Terminal (с 2026-07-17)

Взят с присланного владельцем референс-скриншота: GNOME Terminal в Ubuntu эпохи Unity.
Баклажановый фон + палитра Tango + Ubuntu Mono.

```css
--color-term-bg: #300a24;      /* фирменный баклажан Ubuntu */
--color-term-head: #26071c;
--color-term-line: #4a1839;
--color-term-text: #eeeeec;
--color-term-dim: #a1889b;
--color-term-prompt: #8ae234;  /* яркий зелёный Tango */
--color-term-red: #ef2929;
--color-term-blue: #729fcf;
--color-term-cyan: #34e2e2;
--color-term-yellow: #fce94f;
--font-term: var(--font-ubuntu-mono), ui-monospace, "SF Mono", Menlo, Consolas, monospace;
```

- **За:** мгновенно узнаваемо, аутентично, есть характер; значения Tango — ровно убунтовский
  `LS_COLORS`, поэтому вывод в стиле `ls` можно рисовать достоверно.
- **Против:** баклажановый фиолетовый — сильный акцент, уводящий от нейтрально-чёрной гаммы сайта
  с изумрудным акцентом; плюс он привязывает визуальную идентичность платформы к одному дистрибутиву.
- **Открытый вопрос:** конкурирует ли фиолетовый с зелёным акцентом сайта на страницах, где
  терминал стоит рядом с изумрудным UI? В масштабе пока не оценивалось.

### Альтернатива A — нейтральный чёрно-серый (прежний выбор, 2026-07-16 → 2026-07-17)

Исходный вид. Намеренно нейтрален по палитре, чтобы терминал не конкурировал с сайтом.

```css
--color-term-bg: #0d0d0d;
--color-term-head: #191919;
--color-term-line: #2a2a2a;
--color-term-text: #d4d4d4;
--color-term-dim: #6f6f6f;
--color-term-prompt: #34d399;  /* изумруд сайта */
--color-term-red: #f87171;
--color-term-blue: #60a5fa;
--color-term-cyan: #22d3ee;
--color-term-yellow: #fbbf24;
--font-term: ui-monospace, "SF Mono", Menlo, Consolas, monospace;
```

- **За:** сливается с палитрой сайта; изумрудный промпт перекликается с брендовым акцентом.
- **Против:** обезличенно — читается как «код-блок с шапкой», а не как терминал.

### Альтернатива B — палитра Tango на чёрном (не собиралась)

Средний путь, который предлагался и не был выбран: оставить `#0d0d0d`, взять цвета Tango и Ubuntu
Mono. Берём `bg`/`head`/`line`/`dim` из альтернативы A и `text`/`prompt`/`red`/`blue`/`cyan`/
`yellow`/`--font-term` из текущего выбора. Стоит попробовать, если баклажан окажется слишком громким.

### Альтернатива C — Ubuntu + аутентичная шапка Ambiance (не собиралась)

Текущий выбор сохраняет нейтральную шапку с тремя точками. Максимальная верность референсу
заменила бы её на оформление Ambiance: кнопки окна **слева**, заголовок вида `user@host: /` и
полоса вкладок. Стоит правки компонента (не только токенов) и усиливает декоративность.

## Развилка 2 — шрифты

Два независимых выбора, которые взаимодействуют: шрифт **сайта** и шрифт **терминала**.

### 2a — шрифт сайта. Текущий выбор: IBM Plex Sans (с 2026-07-16)

Подключён в `src/app/[lang]/layout.tsx` через `next/font/google`, сабсеты `latin`+`cyrillic`,
начертания 400–700, выставлен как `--font-plex-sans` и смаплен в `globals.css` на `--font-ui` и
`--font-reading` (чтение теории — по замыслу гротеск).

| Вариант | Состояние | Заметка |
|---|---|---|
| **IBM Plex Sans** | текущий | Нейтральный, технический, отличная кириллица. Чтение и UI на одной семье. |
| Inter | альтернатива | Ещё нейтральнее; характера меньше. |
| Ubuntu Sans | альтернатива | Связал бы сайт с терминалом — целостно, но переводит на Ubuntu-вид всю платформу, а не только терминал. |
| Разделить UI/чтение | не исследовано | `--font-ui` и `--font-reading` — намеренно разные токены: антиква для чтения возможна без правки UI. |

### 2b — шрифт терминала. Текущий выбор: Ubuntu Mono (с 2026-07-17)

Подключён через `next/font/google`, сабсеты `latin`+`cyrillic` (**обязательно** — комментарии в
терминалах пишутся и по-русски), выставлен как `--font-ubuntu-mono` → `--font-term` → утилита
`font-term`.

| Вариант | Состояние | Заметка |
|---|---|---|
| **Ubuntu Mono** | текущий | Аутентичен Ubuntu-виду; узкие глифы. |
| IBM Plex Mono | альтернатива | **Объединил бы типографику** — одна семья со шрифтом сайта. Главный претендент, если Ubuntu-вид отменят. |
| JetBrains Mono | альтернатива | Шире, очень разборчив, хорошая кириллица; больше «IDE», чем «терминал». |
| Fira Code / Cascadia Code | не исследовано | Лигатуры — для коротких учебных фрагментов скорее шум. |
| системный `ui-monospace` | альтернатива | Нулевая сетевая стоимость; вид разный на разных ОС, что вредит элементу, который должен быть единообразным. |

**Открытое напряжение, которое стоит разрешить:** сайт на IBM Plex Sans, а терминал на Ubuntu
Mono — две несвязанные семьи. Либо это читается как намеренный контраст (терминал — отдельный
артефакт), либо как несогласованность. Пока не оценено.

**Подводный камень — размер связан со шрифтом.** У Ubuntu Mono глифы уже, поэтому терминал был
поднят с `13px` до `14px` для оптического паритета. **При любой смене шрифта терминала размер надо
перепроверить**, иначе текст будет выглядеть не в масштабе.

## Смежные развилки, отслеживаемые отдельно

- **Оттенок зелёного акцента** — `docs/green-accent-picker.html` (интерактивный подборщик с
  превью сайта; в приоритете был Sage). По-прежнему открыта.
- **Тема** — `terminal` (тёмная, по умолчанию) против `saas` (светлая), блок `[data-theme="saas"]`
  в `globals.css`. Обе поставляются; это не развилка на разрешение, но изменения здесь
  взаимодействуют с инвариантом развилки 1 «тёмный в обеих темах».

## Практическая заметка

Так как развилка 1 — чистые токены, примерка дизайна выглядит так: заменить блок в `globals.css`
→ `rm -rf .next` → перезапустить dev. **Шаг с кэшем обязателен** — Turbopack кэширует CSS-переменные,
и правки в `@theme` тихо не применяются, пока кэш не очищен.
