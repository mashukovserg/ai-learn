# Режим Roadmap View

Референс-стиль для будущих экранов AI Learn Platform, где нужно показывать траектории, лестницы роста или ветвящиеся пути.

Использовать этот режим для:
- карьерных карт
- лестниц роста навыков
- обзоров путей с развилками
- экранов эволюции ролей

Не использовать этот режим для:
- длинных reading-first глав теории
- обычных карточек дашборда
- плотных сравнительных таблиц

## Базовое ощущение

Режим должен ощущаться как:
- редакционный
- структурный
- спокойный
- технический
- собранный

Визуальная цель: темный low-glare roadmap-экран с сильной иерархией информации и почти полным отсутствием декоративного шума.

## Анатомия layout

Экран строится из пяти слоев:

1. Верхний eyebrow-label.
   Пример: `CAREER ROADMAP`

2. Крупный титульный блок.
   Допустим заголовок в две или три строки, если так лучше работает ритм.

3. Короткое поясняющее вступление.
   Одна или две строки с левым выравниванием. Без длинного hero-абзаца.

4. Ряд переключателей треков.
   Крупные pill-кнопки вроде `IC Track`, `Research Track`, `Management Track`.

5. Основное stacked-тело roadmap.
   Слева вертикальный timeline, справа стек раскрывающихся карточек этапов.

Ниже основной лестницы могут появляться вторичные блоки:
- карточки развилок
- поддерживающие заметки
- подсказки по переходам

## Правила композиции

- Основной поток чтения должен оставаться одноколоночным.
- Используйте левую временную ось с маркерами лет.
- Для главной лестницы используйте stacked-cards, а не desktop split-columns.
- Сохраняйте широкое горизонтальное дыхание вокруг roadmap.
- Внутри карточек держите композицию компактной и сканируемой.
- На экране должна доминировать одна идея, а не набор конкурирующих панелей.

## Типографика

- Eyebrow и метаданные: моноширинный шрифт, uppercase, широкий tracking.
- Главный заголовок: serif или reading-display стиль, крупный размер, высокий контраст.
- Заголовки карточек: monospaced или технический sans с уверенной насыщенностью.
- Поддерживающий текст: левое выравнивание, приглушенный тон, короткие формулировки.
- Годы и level-маркеры: monospaced, сдержанные, вторичный акцент.

Иерархия типографики должна читаться так:
- display-заголовок
- заголовок карточки
- метаданные уровня / лет
- подписи чипов
- поддерживающий текст

## Цветовое направление

Использовать существующую систему темных токенов платформы:
- фоны из `bg-base`, `bg-card-dark`, `bg-card`
- границы из `border-border-card`, `border-border-subtle`, `border-border-emphasis`

Не вводить новые произвольные hex-цвета для фонов и границ.

Акценты использовать ограниченно:
- timeline-dot может иметь свой акцент
- chips могут иметь мягкие пастельные заливки
- активная кнопка трека может отличаться чуть более сильной границей и контрастом текста

Палитра должна оставаться приглушенной:
- угольная база
- теплый off-white текст
- мягкие mint / violet / amber chips
- низкоконтрастные разделители

Избегать:
- неонового свечения
- шумных градиентов
- тяжелых теней
- насыщенных брендовых плашек

## Система карточек

Каждая карточка этапа roadmap должна содержать:
- название роли
- диапазон уровня
- диапазон лет
- компактный ряд chips
- опциональный скрытый раскрывающийся контент

Поведение карточек:
- collapsed by default допустим
- в полном виде может быть только активный или выбранный этап
- раскрытие должно добавлять глубину, а не дублировать summary

Тон карточек:
- скругленные, но не игривые
- достаточно плотные, чтобы ощущаться как console-style dashboard
- визуально спокойные

## Chips

Chips в этом режиме - это функциональные метки, а не декор.

Хорошее содержимое chips:
- ключевые навыки
- инструменты
- зоны ответственности
- сигналы масштаба роли

Стиль chips:
- светлая пастельная заливка
- темный текст
- pill-радиус
- плотный вертикальный padding
- компактные промежутки между chips

Хорошие категории chips:
- systems focus
- research depth
- people leverage
- infra
- evaluation
- product translation

## Timeline

Левая ось критически важна. Она должна объяснять прогрессию еще до чтения абзацев.

Правила timeline:
- одна вертикальная линия
- один маркер на этап
- приглушенные подписи лет
- достаточно воздуха между шагами
- маркеры должны чисто выравниваться относительно карточек этапов

## Взаимодействие

- Переключение треков должно ощущаться мгновенно и стабильно.
- Раскрытие должно быть сдержанным, не театральным.
- Hover-состояния должны быть минимальными.
- Motion должен помогать ориентироваться, а не развлекать.

Предпочтительное поведение:
- быстрая смена контента
- без больших layout-jumps
- без bouncing-анимаций
- без пульсирующих attention-seeking эффектов

## Стиль текста

Текст в этом режиме должен быть:
- коротким
- аналитичным
- практическим
- roadmap-ориентированным

Избегать:
- хайпа
- мотивационных клише
- драматичного продуктового языка
- курсивных целых абзацев

## Лучшие сценарии использования в AI Learn

Этот режим особенно уместен для:
- AI career trajectories
- роста роли по seniority
- карт развилок после базовых комнат
- экранов выбора пути
- лестниц зрелости навыков

## Антипаттерны

Нельзя превращать этот режим в:
- глянцевый marketing-hero
- дашборд с множеством несвязанных виджетов
- двухколоночную narrative-статью
- grid карточек без логики прогрессии
- светящуюся cyberpunk-панель

## Примечания по реализации в этом репозитории

- Сохранять двуязычный контент (`en` и `ru`).
- Переиспользовать текущие темные theme tokens.
- Держать body-текст с левым выравниванием.
- Заголовки оставлять текстовыми, без leading-icons.
- Предпочитать явные data-driven описания этапов.
- Этот режим должен сосуществовать с текущим reading-first режимом теории, а не заменять его глобально.
